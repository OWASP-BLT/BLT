# BLT Page Relevance Ratings

Based on the forum discussion at https://github.com/orgs/OWASP-BLT/discussions/5495

## Rating Scale
- **1-20**: Core features - Essential to the product's main purpose
- **21-40**: Important secondary features - Valuable but not core
- **41-60**: Useful features - Nice to have
- **61-80**: Peripheral features - Questionable value
- **81-100**: Remove or significantly reconsider - Not aligned with core focus

## Core Focus Areas (from forum):
1. **Vulnerability Discovery** - Finding and reporting unpatched CVEs
2. **UI/UX Bug Discovery** - Internet-wide bug reporting
3. **Education & Contributor Growth** - Training and onboarding
4. **Incentivizing Fixes** - Reward systems for security PRs
5. **Knowledge Sharing** - Community learning without publishing raw vulnerabilities
